## 二、供应商精简（只保留 Qwen + Doubao）

### 2.1 核心思路

v1.0.x 支持 8 家供应商（doubao, qwen, tencent, kimi, xiaomi, yi, baidu, custom），但实际已验证可用的**原生多图融合**能力只存在于 **doubao（字节豆包 Seedream 5.0）** 和 **qwen（阿里 Qwen-Image 2.0 Pro）**。

**决策**：只保留 Qwen + Doubao，其余 6 家供应商及对应的双阶段配对模式全部移除。

### 2.2 关于 generateAsset（AI 生成素材）的保留决策

经查阅官方文档确认：
- **Qwen-Image 2.0 Pro**：支持 `multimodal-generation/generation` 端点，content 数组只传 `{ text: prompt }` 即可做纯文本到图片的生成（T2I）
- **Doubao Seedream 5.0**：使用 `/images/generations` 端点，不传 `image` 字段即为标准 T2I

结论：**generateAsset 保留**，但重写 T2I 路由为只走 Qwen/Doubao 的原生接口。

### 2.3 具体修改

#### `types.ts`

- `ApiProvider`：从 8  种 → `'doubao' | 'qwen'`
- `ApiFormat`：从 5 种 → `'openai' | 'dashscope'`
- 删除 `CustomProviderConfig` 接口
- `ProviderCapabilities`：精简字段，`vision` / `streaming` / `tokenPlan` 等不再需要
- `RuntimeSettings`：从 4 个字段简化为 `{ currentProvider: ApiProvider }`
- `ApiSettings`：`providers` 从宽泛的 `Record<ApiProvider, ProviderConfig>` 简化为 `{ doubao: ProviderConfig; qwen: ProviderConfig }`

| `buildMultimodalMessages` | 双阶段消息构建移除 |
| `generateImageFn` | 通用 T2I 路由移除 |
| `testProviderConnection` | 测试连接功能移除 |
| `getProviderConfig` | 不再需要按 provider 查找配置 |

**保留 + 重写的函数：**

| 函数 | 调整方式 |

**保留不动：**

## 七、我对 v1.1.0 的补充意见

### ✅ 关于 generateAsset 的分析结果

查阅 Qwen-Image 2.0 Pro 和 Doubao Seedream 5.0 官方文档后确认：

| 模型 | 原生融合 | 文生图（T2I） | 接口是否兼容 |
|------|---------|--------------|------------|
| Qwen-Image 2.0 Pro | ✅ `multimodal-generation/generation` | ✅ content 只传 `{ text }` | 同一端点，不同入参 |
| Doubao Seedream 5.0 | ✅ `/images/generations` + `image` | ✅ `/images/generations` 无 `image` | 同一端点，省略字段 |

**结论**：generateAsset 可以保留，只需要为每个 provider 的 T2I 模式写一个小分支。改动量很小。

### 🏷️ 版本命名建议

建议在 README 和页面标注 **v1.1.0 — 精简稳定版**，明确与多供应商试验期的分界。

### 🔮 未来扩展

`ApiProvider` 直接写死 `'doubao' | 'qwen'` 即可，不必过度设计。将来加供应商时再扩类型。
